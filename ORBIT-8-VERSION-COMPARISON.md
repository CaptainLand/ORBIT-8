# ORBIT-8 v1-v1.7 横向性能对比报告

生成日期：2026-07-02

## 结论摘要

- **当前推荐默认版本：v1.7.1。** 本表的主列截止 v1.7；v1.7.1 进一步修正配置定义并限制纵连。
- **v1 在 B.M.S. 单曲 F1 仍略高。** v1 为 0.6409，v1.7 为 0.6378；v1 更保守，而 v1.7 召回更高、配置更丰富。
- **v1.5 是采音泛化的第一次明显提升。** 验证集最终落键 F1 从 v1 的 0.334 提升到 0.432，约增长 29.3%。
- **v1.6 的核心提升在排键而非采音。** 它首次显式学习交互、扫键和纵连，并通过四向镜像训练消除固定方向偏置。
- **v1.7 当前主要短板是密度校准。** B.M.S. 比官谱多 32 个事件，说明高召回仍会带来轻微过采。

## B.M.S. 统一基准

基准为训练集中的 B.M.S. 12.6 官谱：436 个事件、322 个唯一音头。四版均解析最终 `maidata.txt`，采用一对一贪心匹配，容差 ±30ms。

配置统一采用 v1.7.1 严格定义：交互=16分交替，纵连=同键16分，扫键=16/24/32分连续位移，8分不属于任何配置。

### B.M.S. 解读

- v1 的 F1 略高来自更保守的音头数量和更高精度，但它没有识别出交互，Hold 数也明显偏低。
- v1.5 提高召回和 Hold 比例，但排键仍依赖逐 Note 位移预测，没有完整配置概念。
- v1.6 对音没有变化，因为它沿用 v1.5 采音器；提升主要体现为首次出现交互段。
- v1.7 命中音头最多，交互段与 v1.6 持平，但事件数比官谱高 7.3%，下一步应校准难度对应密度。

v1.7 在候选阶段就阻止孤立高速音，因此不需要在生成结束后大量删 Note。只有音频起音头连续四次高置信命中的 24/32 分段能够保留，并会被排成扫键。

## 限制与公平性

1. B.M.S. 存在于训练数据中，因此该表衡量版本行为和复现能力，不代表陌生歌曲泛化。
2. v1 使用 199.978 BPM，后续版本使用 200 BPM；差异很小，但不是完全相同输入。
3. v1/v1.5 使用的训练集规模和验证划分不同，验证 F1应视为趋势而不是严格竞赛排名。
4. 自动指标不能衡量完整谱面审美、手感和采音情绪，仍需 Visual Maimai 实机审查。

## 建议

- 日常生成使用 **v1.7**。
- 保留 v1 作为“保守采音参考”，用于检查 v1.7 是否过采。
- 下一版本优先做难度密度校准和真正未见歌曲的固定测试集，而不是继续扩大模型。
- 建议固定 20-30 首未参与训练的歌曲，长期记录 ±20/30/50ms F1、配置段、KPS和人工评分。

原始机器可读结果见 `ORBIT-8-VERSION-COMPARISON.json`。
